# Forms Components

Forms components are the building blocks for user input and data collection in your application. They provide consistent styling, behavior, and accessibility across all form elements.

## Available Components

### Button
- Primary action trigger
- Multiple variants (primary, secondary, outline, ghost, danger)
- Support for icons and loading states
- Full keyboard accessibility

### Input
- Text input fields
- Support for labels, helper text, and error states
- Icon support (left and right)
- Various sizes and states

## Common Features

All form components include:
- Consistent styling and theming
- Keyboard navigation support
- Screen reader compatibility
- Focus management
- Error handling
- Loading states where applicable

## Best Practices

1. **Accessibility**
   - Always provide labels
   - Use appropriate ARIA attributes
   - Ensure keyboard navigation
   - Maintain sufficient contrast

2. **Validation**
   - Provide clear error messages
   - Use appropriate input types
   - Validate on appropriate events
   - Show loading states for async validation

3. **User Experience**
   - Keep forms simple and focused
   - Group related fields
   - Provide clear feedback
   - Use appropriate input types

## Theming

Forms components use the following theme variables:
- `--color-primary`: Primary actions and focus states
- `--color-error`: Error states and messages
- `--color-text`: Text colors
- `--color-border`: Input borders
- `--color-background`: Input backgrounds
